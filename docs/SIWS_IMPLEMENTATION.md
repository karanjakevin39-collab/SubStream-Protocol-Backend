# Sign-In With Stellar (SIWS) Implementation

This document describes the SEP-10 compliant Sign-In With Stellar implementation for the SubStream Protocol Backend.

## Overview

The SIWS implementation allows users to authenticate using their Stellar wallet by signing a challenge transaction generated by the server. This follows the [SEP-10](https://github.com/stellar/stellar-protocol/blob/master/ecosystem/sep-0010.md) standard for Stellar web authentication.

## Architecture

### Components

1. **StellarAuthService** (`services/stellarAuthService.js`)
   - Handles SEP-10 challenge generation and verification
   - Manages challenge lifecycle and expiration
   - Validates Stellar account status

2. **Stellar Auth Middleware** (`middleware/stellarAuth.js`)
   - JWT token generation and validation
   - Session management with HttpOnly cookies
   - Wallet switching functionality
   - Session invalidation for inactive/merged accounts

3. **Stellar Auth Routes** (`routes/stellarAuth.js`)
   - RESTful endpoints for authentication flow
   - Challenge generation and verification
   - Session management endpoints

## API Endpoints

### Authentication Flow

#### 1. Generate Challenge
```
GET /auth/stellar/challenge?publicKey=<STELLAR_PUBLIC_KEY>
```

**Response:**
```json
{
  "success": true,
  "challenge": "base64-xdr-transaction",
  "nonce": "random-nonce",
  "expiresAt": "2024-01-01T12:00:00.000Z"
}
```

#### 2. Verify Challenge and Login
```
POST /auth/stellar/login
Content-Type: application/json

{
  "publicKey": "G...",
  "challengeXDR": "signed-transaction-xdr"
}
```

**Response:**
```json
{
  "success": true,
  "token": "jwt-token",
  "user": {
    "publicKey": "g...",
    "tier": "bronze",
    "type": "stellar"
  },
  "expiresIn": 86400
}
```

### Session Management

#### Get Session Info
```
GET /auth/stellar/session
Authorization: Bearer <JWT_TOKEN>
```

#### Switch Wallet
```
POST /auth/stellar/switch
Authorization: Bearer <JWT_TOKEN>

{
  "newPublicKey": "G...",
  "challengeXDR": "signed-challenge-for-new-wallet"
}
```

#### Logout
```
POST /auth/stellar/logout
Authorization: Bearer <JWT_TOKEN>
```

### Utility Endpoints

#### Challenge Status
```
GET /auth/stellar/challenge-status?publicKey=<STELLAR_PUBLIC_KEY>
```

#### Validate All Sessions (Admin)
```
POST /auth/stellar/validate-sessions
```

## Security Features

### 1. SEP-10 Compliance
- Challenge transactions with time bounds (5 minutes expiration)
- Domain-specific manageData operations
- Cryptographic signature verification

### 2. Session Security
- HttpOnly cookies to prevent XSS attacks
- Secure flag in production
- SameSite protection against CSRF
- JWT tokens with expiration

### 3. Account Validation
- Automatic verification of account status
- Session invalidation for merged/inactive accounts
- Prevention of wallet reuse after account changes

### 4. Wallet Switching
- Secure wallet switching with challenge verification
- Automatic session invalidation for old wallet
- Continuous user tier preservation

## Client Integration

### JavaScript Example

```javascript
// 1. Generate challenge
const response = await fetch('/auth/stellar/challenge?publicKey=' + publicKey);
const { challenge } = await response.json();

// 2. Parse and sign challenge (using stellar-sdk)
const transaction = new StellarSdk.Transaction(challenge, networkPassphrase);
transaction.sign(StellarSdk.Keypair.fromSecret(secretKey));
const signedChallenge = transaction.toXDR();

// 3. Authenticate
const authResponse = await fetch('/auth/stellar/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    publicKey,
    challengeXDR: signedChallenge
  })
});

const { token, user } = await authResponse.json();
```

### React Hook Example

```javascript
function useStellarAuth() {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);

  const login = async (publicKey, signedChallenge) => {
    setLoading(true);
    try {
      const response = await fetch('/auth/stellar/login', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          publicKey,
          challengeXDR: signedChallenge
        })
      });
      
      const { token, user } = await response.json();
      localStorage.setItem('stellar_token', token);
      setUser(user);
    } catch (error) {
      console.error('Login failed:', error);
    } finally {
      setLoading(false);
    }
  };

  const logout = async () => {
    try {
      await fetch('/auth/stellar/logout', {
        method: 'POST',
        headers: { 
          'Authorization': `Bearer ${localStorage.getItem('stellar_token')}`
        }
      });
    } finally {
      localStorage.removeItem('stellar_token');
      setUser(null);
    }
  };

  return { user, login, logout, loading };
}
```

## Troubleshooting

### Common Issues

1. **Invalid Challenge XDR**
   - Ensure the challenge is signed with the correct keypair
   - Verify the network passphrase matches the server configuration

2. **Expired Nonce**
   - Challenges expire after 5 minutes
   - Generate a new challenge if needed

3. **Account Not Found**
   - Verify the account exists on the configured network
   - Check network configuration (testnet vs mainnet)

4. **Session Invalidated**
   - Sessions are invalidated if the account becomes inactive or merged
   - Users must re-authenticate in this case

### Debug Mode

Enable debug logging:
```bash
DEBUG=stellar:* npm run dev
```

## Migration from SIWE

If migrating from Sign-In with Ethereum:

1. Update client-side authentication flow to use Stellar SDK instead of ethers.js
2. Replace Ethereum address validation with Stellar public key validation
3. Update API endpoints from `/auth/nonce` and `/auth/login` to `/auth/stellar/challenge` and `/auth/stellar/login`
4. Handle the different response formats and XDR-based challenges
